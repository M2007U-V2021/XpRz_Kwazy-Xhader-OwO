Color Transform

a Color Transform is refered to as a function such that it takes a color as an input, then outputs a color

note

here we will refer "input Color" as inColor and "output Color" as outColor

Here we will go into the details of every color Transform operation we have

Invert​

each color channel can be inverted separately, including the aplha chanel.
each slider will LERP between the original color channel strength and the inverted color channel strength

what is LERP ?

LERP is a famous function used throughout many industries and scenarios, it takes 3 inputs : Starting Object, Ending Object, T-value based on the T-value, the LERP function will give you a blend of the Starting Object and the Ending Object

LERP(Start,End,t) = (End - Start) * t + Start

or most of the time how they would write it

LERP(P0,P1,t) = (P1-P0)t + P0 = P0(1-t) + P1(t)
tip

when all T values (or sliders) are crancked to 1, this inverts all channels, which is the familiar "invert" effect we all know and love, just in case, remember to let the alpha channel stay the same

Tween Animation​

parameters : fixed4 inColor0, fixed4 inColor1, fixed4 inColor2, fixed4 inColor3, vector Durations

in KXOwO, you can animate colors without using Unity Animation,

note

if you didn't see that the colors are changing in Edit mode in Unity, you can add a Particle System to let it act as a clock.

The maximum keyframe of colors we can have here is 4, thats why we have 4 colors to choose by default, the Shader will go back to the first frame when all 4 frames has been displayed,
i.e. it loops

the vector Durations will determine the duration between the 2 keyframes

However, if you plan to have only one color, you can type in the first duration chunk "-1", this will tell the shader that we are using inColor0 only.

Threshold​

sometimes, colors needs to go through some threshold tests, if they fullfill the test, they will be unaltered, else they will be overriden by some other values. here we are going to focus on one single channel

parameters : float ValueLow, float ThreshLow, float ThreshHgh, float ValueHgh

when input < ThreshLow, then output = ValueLow, when input > ThreshHgh, this output = ValueHgh, otherwise, output = input

caution

Threshold will only work when ThreshLow is smaller than ThreshHgh, otherwise, output = input.

JUST IN CASE

ValHgh doesn't need to be larger than ThreshHgh,
ValLow doesn't need to be lesser than ThreshLow, But this may cause unexpected results,
only do this if you have a good reason of doing so.

HSV adjust​

Just like in other popular softwares (like photoshop, SAI, etc), here we can manipulate the texture by shifting the Hue, Saturation, and Vibrance values.

Here we have a hue shift of 0.0 (original, untouched), 0.1, 0.2, and 0.3

If this is your first time seeing this thing, here is an short explanation for HSV :

A color can be described in many ways, RGB is one of them, YMCK is another one of them, HSV is one of them as well.

shortformWhat it isDesc
HHueHere we plot the spectrum of colors on a circle, from 0 degrees to 360 degrees. With Red being 0 deg, Green as 120 deg, blue as 240 deg. A Hue of a color is the angle of the color on this circle.
SSaturationHow pure the color is, if a color has a high saturation, it is considered to be more "clean", if the saturation is low, then it is more "grey-ish"
VVibrancehow bright the color is, when V=0, it is totally dark, when V=1, it is bright
caution

Saturation and Vibrance are totally different, heres a table to show how they are separated parameters :

0<V<1total Greynot that Grey, but not that Purevery Pure, but a bit dark
V=1total Whitea pale version of the colorComplete Perfect of the Color
