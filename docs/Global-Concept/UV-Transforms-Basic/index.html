UV Transform - Basics

What is a UV Transform ?​

a UV Transform is a function that takes a UV coordinate and returns a new UV coordinate.

Different operations will cause different effects

In Other Words

UVTransform(inUV,Operation) = outUV

Example

some examples that we usually see are "Shifting", "Rotation", "Scalling"

Chained UV Transform​

since a single UVTransform can be treated as a function, we can have multiple UVTransform functions chained in a line

Each Operation is associated with an ID, the ID is used to tell the shader what operation it is.

Example
  • "Shift" has an ID "12"
  • "Rotation" has an ID "13"
  • "Scalling" has an ID "14"

Here's a screenshot of how a chain of UVTransform might look like on a Unity Inspector

Each Transform Vec Vector (Twansfowm Vec N) will hold the parameters of a UV Transform Chain segment, but without defining which Operation it is, a Transform Vector by itself is useless.

In this case we will use Transform Mode 0 Vector and Transform Mode 1 Vector to tell the shader what are the operations for each Transform Vec Vectors

Let's say you want TransformVec4 to have the operation "Rotation" (in KXOwO, the operation "Rotation" has an ID "12"), you will type "12" into TransformMode1.x

Example 2

Just In Case

different operation order will cause different effects,
"a rotation then a shift" is different from "a shift then a rotation"

In most other popular shaders, they provide a fixed operation order transform system, like Unity Standard Shader, Poiyomi Shaders, Mochie Shaders, etc

Types of UVTransforms​

in KXOwO, we have 3 types of Operations :

  • Regular
  • Collapse
  • Chaotic

Regular Operation​

if an operation is Regular, then it maps all inUVs to all outUVs such that no outUV will have multiple inUVs meaning you can find a reverse operation to undo the work

Example

to make things simple, let's build an operation :

Plus(input,amount) = output
such that
output = input + amount

if amount = 3, then we will have

input0123n
output3456n+3

if we are given an output = 18
then we can derive that 

output = input + amount
output - amount = input
18 - 3 = input
15 = input

here we can get 15 back from 18, therefore this operation is regular

Collapse Operation​

A Collapse Operation will collapse multiple inUVs onto the same outUVs, and thus when given the outUV, we are unable to reconstruct the orignal inUV

Example

let make a collapse operation
Remainder(input,divisor) = output
such that
output = input % divisor

if divisor = 3, then we can have :

input0123456789011123n3n+13n+2
output0120120120120012

as you can see, 0,3,6,9,...,3n have been mapped to 0, If you are given 0 as the output, you are unable to find what is the original input, because it could be 0,3,6,9,...,3n

Chaotic Operation​

an operation is Chaotic if a small change for the input will cause a huge change in the output.
Also this type of operation is meant to generate patterns/output that looks randomized (but they are actually not randomized)

Example 1

Usually a pseudo random number generator is a nice example, if the given seed is different, then the results it is going to output will be totally different.

Example 2

hashing algorithyms take a string as an input, jumbled it up in some way (inputs are passed by value by default), then gives you an output.

If you give the algorithym the same input again, it will output the same output, but if you change a single bit in the input, the output will be drammatically different.

say we are using SHA256, and we have 2 inputs "hewo" and "newo", you can see how different the outputs are although the inputs only has a 1 letter difference.

Just In Case

just because an operation is chaotic, it doesn't mean it is a collapse operation as well, a chaotic operation might can also be regular
