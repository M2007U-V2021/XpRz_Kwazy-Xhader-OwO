Externel - Vector Color

the main concept is : the more the face facing at a specific direction (world space), to more it is going to get colored

In KXOwO, we have 4 sets of these, meaning you can have atmost 4 vector colors

Here's a screenshot of a sphere having 3 Vector Colors with 3 different Vectors:

Vector ColorRGBAXYZ
VC1(1,0,0,1)(0,1,-1)
VC2(0,1,0,1)(0,1,1)
VC3(0,0,1,1)(1,0,-1)

Funny Stuff (LMAO)

This feature was added originaly to fake lightning, but now KXOwO can react to realtime lights.
However, this is still usefull to fake bakedLightning

To activate it

Since Vector Color is an External Layer, this means that at least one of the Texture Layer needs to be influence in order to see the Vector color in action. (i.e. just by the Vector Color Alone you will see nothing). In this case, find a Texture Layer that you want it to be influenced, then crank up the slider Vektor Kolor Infwuence

Force Alpha

under Main Texture Layer, where you adjust how much the Layer is influenced by Vector Color, there is another slider Vektor Kolor ForceAfwa, this will make the Vektor Color result be transparent or opaque

Direction Vector​

This will decide the direction that the mesh faces needs to face at in order to get colored

here's a series of snapshots with the vector X, Y, Z

Wamp / Ramp​

the exponent for the Vector's dot product

In other words

Generally
VectorColor(;N) = dot(GivenVector(;N),MeshNormalVector)^Ramp

VectorColorTotal = VectorColor0 + VectorColor1 + VectorColor2 + VectorColor3

here's a series of snapshots of the Ramp parameter being 0.0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0

Strength​

this will determine how strong the Vector Color is. Here's a series of screenshots with the Strengh being 0.1, 0.2, 0.3, 0.4

Mask​

Each Vector Color can me masked saperately. For more info, refer to the document : 02_Fragment-Shader/01_ForwardBase/03-General-Mask
